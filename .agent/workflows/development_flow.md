# 3D Shape Slicer 開発フロー

このプロジェクトでは、不具合や予期せぬ挙動を防ぐため、以下の手順で開発を行います。

## 1. テスト環境での修正 (`apps/3D Shape Slicer/test/`)

1. 修正が必要な場合、まずは `apps/3D Shape Slicer/test/` 内のファイル（`index.html`, `js/app.js` 等）を編集します。
2. 修正後、ブラウザ等で動作確認を行い、意図した通りに動くことを確認します。

## 2. 本番環境への反映 (`apps/3D Shape Slicer/`)

1. テスト環境での動作が確認できたら、その修正内容を本番環境のファイルに反映します。
2. 単純なコピーではなく、パスの違い（`test/js/app.js` → `js/app.js` など）に注意してください。

## 3. 更新履歴の記録

1. 本番環境への反映が完了したら、必ず `sakura-academia/UPDATE_LOG.md` に更新内容を記録します。
2. 記録の際は、テスト環境での検証が完了した旨も併記するとより望ましいです。

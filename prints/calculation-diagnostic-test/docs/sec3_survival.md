---

## テストの目的

第3部「持久力・整理力」の7単元（A～G）から各レベル1問ずつ計35問を出題し、長時間の計算耐久力と情報整理力を測定する。

### テスト形式

- **問題数**: 35問（7単元 × 5レベル）
- **制限時間**: 3分
- **採点方法**: 番号順に解き、連続正解数が得点（Q1から何問連続で正解したか）
- **セット数**: 3セット（セットA, B, C）

### 判定基準

| 得点 | ランク | 状態 |
|------|--------|------|
| 30～35問 | S | 完全自動化・整理力完璧（Mastery） |
| 20～29問 | A | 流暢性・整理力良好（Fluency） |
| 10～19問 | B | 手順的知識（Procedural） |
| 0～9問 | C | 未習熟（Pre-structural） |

## セットB・Cの設計方針（実装時）

### セット間の多様性

- **異なる数値**: セットAで「□+7=15」ならセットBは「□+8=17」
- **異なる問題**: セットAで「□×3/5=9/10」ならセットBは「□×2/7=5/14」
- **同じ難易度**: レベルは統一（Level 1はどのセットも簡単）

---

## 避けるべきパターン（実装時）

### ❌ 計算が複雑すぎる

```
NG例: □×7/13=49/169
理由: 分数が複雑すぎる。基本的な分数に限定。
```

### ❌ 単位変換が難しすぎる

```
NG例: 3.7km² = □m²
理由: 面積の単位変換は小学生には難しい。
```

**テスト形式**: 連続正解チャレンジ 35問

---

## テストの目的

第1部「反射神経」の7単元（A～G）から各レベル1問ずつ計35問を出題し、基本的な計算の自動化レベルを測定する。

### テスト形式

- **問題数**: 35問（7単元 × 5レベル）
- **制限時間**: 3分
- **採点方法**: 番号順に解き、連続正解数が得点（Q1から何問連続で正解したか）
- **セット数**: 3セット（セットA, B, C）

### 判定基準

| 得点 | ランク | 状態 |
|------|--------|------|
| 30～35問 | S | 完全自動化（Mastery / Automatic） |
| 20～29問 | A | 流暢性（Fluency） |
| 10～19問 | B | 手順的知識（Procedural） |
| 0～9問 | C | 未習熟（Pre-structural） |

## セットB・Cの設計方針

### セット間の多様性

- **異なる数値**: セットAで「0.5→1/2」ならセットBは「0.25→1/4」
- **異なる問題**: セットAで「11×11」ならセットBは「12×12」
- **同じ難易度**: レベルは統一（Level 1はどのセットも簡単）

### 例：Q1の3セット比較

| セット | 単元A（Q1） | 答え | 難易度 |
|--------|------------|------|--------|
| セットA | 0.5 → □（既約分数） | 1/2 | Level 1 |
| セットB | 0.25 → □（既約分数） | 1/4 | Level 1 |
| セットC | 0.75 → □（既約分数） | 3/4 | Level 1 |

## 避けるべきパターン

### ❌ 逆問題の配置

```
NG例:
  Q1: 0.5 → 1/2
  Q8: 1/2 → 0.5
理由: 逆の問題だと答えが簡単にわかってしまう
対策: 別の数値を使う（Q8を「1/8→0.125」にするなど）
```

### ❌ 同じ答えの問題

```
NG例:
  Q5: 25×4 = 100
  Q6: 32+68 = 100
理由: 同じ答えだと記憶に頼れてしまう
対策: 一方を別の数値に変更（Q6を「43+57=100」など）
```

### ❌ ○×判定の偏り

```
NG例:
  Q4: 51は素数か？ → ×
  Q11: 119は素数か？ → ×（両方×）
理由: パターンが読めてしまう
対策: ○と×を混在させる
```

### ❌ 難易度順の乱れ

```
NG例:
  Q22（Level 4）が Q15（Level 3）より簡単
理由: 連続正解の採点方式に矛盾
対策: 各レベル内でも適切な難易度の問題を選ぶ
```

## 特殊ルール：単元B（平方数）の扱い

### 背景

- 単元B（平方数）はLevel 1-2のみ（小学生向け制限）
- サバイバルテストは35問（7単元×5レベル）必要

### 解決策

- Level 3-5の枠（Q16, Q23, Q30）にも、**単元BのLevel 1-2から問題を抽出**
- 難易度はLevel 3-5相当になるよう、適切な問題を選ぶ

### 具体例

| テスト位置 | 問題 | 出典 | 理由 |
|-----------|------|------|------|
| Q16（Level 3） | 15×15 | sec1b Level 1 | 15の段は暗記の中でも難しめ |
| Q23（Level 4） | 17×17 | sec1b Level 1 | 17の段はさらに難しい |
| Q30（Level 5） | 18×18 | sec1b Level 1 | 18の段は最高難度 |

## 解説文（annotations）の方針

### サバイバルテストは解説なし？

- **表示**: main.htmlでサバイバルテスト（`survival-sheet`クラス）の場合、解説文は非表示
- **理由**: 35問と問題数が多いため、解答ページをコンパクトにする
- **ただし**: JSファイルにはannotationsを記載（攻略パスポートで参照可能）

### annotationsの書き方

```javascript
annotations: [
  "【小数分数A-L1】0.5＝1/2。最も基本的な変換。",
  "【平方数B-L1】11×11＝121。11の段の暗記。",
  "【円周率C-L1】3.14×2＝6.28。暗記で即答。",
  // ...
]
```

- **フォーマット**: 【単元名-レベル】＋解説
- **目的**: どの単元のどのレベルから出題したかを明記

## 印刷レイアウト

### 表面（問題ページ）

- **2列レイアウト**: 左列Q1-17、右列Q18-35

- **ヘッダー**: ステージ名、単元タイトル、セットラベル、名前欄

- **2列レイアウト**: 問題＋解答のみ（解説文なし）
- **解答**: 赤色、太字、14pt
- **コンパクト**: 問題間スペース4mm（訓練より密）

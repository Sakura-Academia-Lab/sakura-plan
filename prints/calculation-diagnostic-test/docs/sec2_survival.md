## テストの目的

第2部「構造視力」の7単元（A～G）から各レベル1問ずつ計35問を出題し、計算の構造を見抜く力を測定する。

### テスト形式

- **問題数**: 35問（7単元 × 5レベル）
- **制限時間**: 3分
- **採点方法**: 番号順に解き、連続正解数が得点（Q1から何問連続で正解したか）
- **セット数**: 3セット（セットA, B, C）

### 判定基準

| 得点 | ランク | 状態 |
|------|--------|------|
| 30～35問 | S | 構造完全把握（Structural Mastery） |
| 20～29問 | A | 構造理解（Relational） |
| 10～19問 | B | 手順的知識（Procedural） |
| 0～9問 | C | 未習熟（Pre-structural） |

## セットB・Cの設計方針

### セット間の多様性

- **異なる数値**: セットAで「34×13+34×87」ならセットBは「27×15+27×85」
- **異なる問題**: セットAで「101×24」ならセットBは「99×36」
- **同じ難易度**: レベルは統一（Level 1はどのセットも簡単）

### 例：Q1の3セット比較

| セット | 単元A（Q1） | 答え | 難易度 |
|--------|------------|------|--------|
| セットA | 34×13+34×87 | 3400 | Level 1 |
| セットB | 27×15+27×85 | 2700 | Level 1 |
| セットC | 56×32+56×68 | 5600 | Level 1 |

## 避けるべきパターン

### ❌ 異なる技法の混在（同じ問題内）

```
NG例:
  Q1: 34×13+34×87（分配法則）
  Q2: 99×36（100に近い数）← OK
理由: 別の単元なので混在してOK。
```

### ❌ 同じ答えの問題

```
NG例:
  Q5: 18×5÷6 = 15
  Q6: 1+2+3+4+5 = 15（両方15）
理由: 同じ答えだと記憶に頼れてしまう
対策: 一方を別の数値に変更
```

### ❌ 難易度順の乱れ

```
NG例:
  Q22（Level 4）が Q15（Level 3）より簡単
理由: 連続正解の採点方式に矛盾
対策: 各レベル内でも適切な難易度の問題を選ぶ
```

## 解説文（annotations）の方針

### サバイバルテストは解説なし？

- **表示**: main.htmlでサバイバルテスト（`survival-sheet`クラス）の場合、解説文は非表示
- **理由**: 35問と問題数が多いため、解答ページをコンパクトにする
- **ただし**: JSファイルにはannotationsを記載（攻略パスポートで参照可能）

### annotationsの書き方

```javascript
annotations: [
  "【分配A-L1】34×13+34×87 = 34×(13+87) = 34×100 = 3400。分配法則の基本。",
  "【分配B-L1】101×24 = (100+1)×24 = 2400+24 = 2424。100に近い数の工夫。",
  "【結合C-L1】25×4 = 100。黄金ペアの基本。暗記必須！",
  // ...
]
```

- **フォーマット**: 【単元名-レベル】＋解説
- **目的**: どの単元のどのレベルから出題したかを明記
